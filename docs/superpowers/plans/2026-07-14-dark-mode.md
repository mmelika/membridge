# Desktop Dashboard Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a System/Light/Dark theme to the Electron dashboard using CSS `light-dark()`, persisted in localStorage, with the neural-graph canvas following the active theme.

**Architecture:** All themed colors live once, as `light-dark(light, dark)` expressions in the "Minimalist Modern" CSS section of `lib/dashboard.js` (and shared vars used by `lib/dashboard-team.js`). Mode switching only flips `document.documentElement.style.colorScheme`. The dead teal-era palette block is removed. Spec: `docs/superpowers/specs/2026-07-14-dark-mode-design.md`.

**Tech Stack:** Vanilla CSS-in-template-string (no build step), Electron 43 Chromium (supports `light-dark()` and `color-mix()`), existing `test/run-tests.js` string-check harness.

```js
function themePref() {
  try {
    var t = localStorage.getItem('mb-theme');
    return t === 'light' || t === 'dark' ? t : 'system';
  } catch (e) { return 'system'; }
}
function applyTheme(pref) {
  document.documentElement.style.colorScheme =
    pref === 'light' || pref === 'dark' ? pref : 'light dark';
  try {
    if (pref === 'light' || pref === 'dark') localStorage.setItem('mb-theme', pref);
    else localStorage.removeItem('mb-theme');
  } catch (e) { /* non-persistent session */ }
}
```

- [ ] **Step 5: Add the Appearance control to the Settings view markup (inside `#view-settings`, alongside the existing model radios), reusing the `.radio` row style:**

```html
<div class="field-label">Appearance</div>
<div id="stTheme">
  <label class="radio"><input type="radio" name="stTheme" value="system"> System</label>
  <label class="radio"><input type="radio" name="stTheme" value="light"> Light</label>
  <label class="radio"><input type="radio" name="stTheme" value="dark"> Dark</label>
</div>
```

And wire it where settings render:

```js
var themeRadios = document.querySelectorAll('input[name=stTheme]');
for (var ti = 0; ti < themeRadios.length; ti++) {
  themeRadios[ti].checked = themeRadios[ti].value === themePref();
  themeRadios[ti].onchange = function () {
    applyTheme(this.value);
    syncThemeSel();
  };
}
function syncThemeSel() {
  var rows = document.querySelectorAll('#stTheme .radio');
  for (var i = 0; i < rows.length; i++) {
    rows[i].classList.toggle('sel', rows[i].querySelector('input').checked);
  }
}
syncThemeSel();
```
